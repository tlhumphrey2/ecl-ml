Introduction to ECL’s Logistic Regression

This document introduces Logistic Regression by talking about A) Logistic _verify_model.ecl and B) Logistic_runtime_on_large_trainingset.ecl – two benchmark programs found in ML.Tests.Benchmarks. These two programs verify the correctness of the model created by Logistic Regression (A, above), and the execution time of Logistic Regression learning on a large training set (B, above).

If you are new to ECL’s Machine Learning (ML) Library and/or HPCC Systems, please read the document, MachineLearning.pdf which is found in the same folder as this document.

Or, if you are a new contributor or thinking about becoming a contributor to the ML Library, please read ContributorsGuide.htm which is also found in the same folder as this document.

Logistic_verify_model.ecl

Logistic_verify_model.ecl verifies the correctness of the model by comparing it to the same created by R’s logistic regression. At the top of the program is an R program, lines 4 through 18, (in comments). It uses the same dataset set as the ECL Logistic Regression. You can cut and paste the R program and drop it into an R window and it will execute.

The inline dataset used for learning is in Lines 31 through 40. In Table 1, below, the dataset is shown with one record (or observation) per line. The record layout that describes each record of this dataset is on lines 24 through 29. Often in machine learning (ML) repositories on the web (e.g. http://archive.ics.uci.edu/ml), you will find datasets in a format similar to that of Table 1, except there is a comma between each field (CVS format, i.e. comma separated vectors). Also very often, the ML repositories won’t have the first column of Table 1, which is a sequential record id (rid) and is needed by the ECL ML macro, ML.ToField (see line 42) to convert the inputted dataset (d) into the form expected by the ECL ML learning algorithms, i.e. the ML.Types.NumericField record layout. In this example, the converted dataset is flds0 (see line 42).

The model creation occurs at line 47 which follows, in gray.

ModelD := LogReg.LearnC(X,Y);

The inputted arguments to LearnC are X, the independent variables (or features) and Y, the dependent variable which is of type ML.Types.DiscreteField. We created X and Y from flds0 in lines 43 through 45. We first, converted flds0 from type ML.Types.NumericField to ML.Types.DiscreteField using ML.Discretize.ByRounding (line 43). Then at line 45 we extracte Y from the discrete dataset. And, the line just before, line 44, we extracte X from flds0.

The created model is outputted at line 47. The model is shown is Table 2, below. The beta values of the model are in the ‘w’ column. The beta ids are in the column labeled ‘number’. And, the beta ids are 0 for the intercept, 1 for the 1st independent variable (in our case age), and 2 for the 2nd independent variable (in our case height). The ‘se’ column gives the standard error for each beta.

The code of lines 50 through 53 is an example of learning with sparse data. It is in this test code to see if we create the same model whether the independent data contains zeros or the zeros are omitted (i.e. the data is sparse). Line 51 removes all zeros from the independent data and therefore creates a sparse dataset. Line 52 creates the model. And, line 53 outputs the model which turns out to look just like the dense model of Table 2, below.

Logistic_runtime_on_large_trainingset.ecl

Logistic_runtime_on_large_trainingset.ecl does learning on 581,012 observations. How much time it takes to create the model is given in the comments at the top of the code, lines 19 through 24.

The training dataset came from the online ML repository, http://archive.ics.uci.edu/ml. It is the forest cover type data which is at https://archive.ics.uci.edu/ml/datasets/Covertype. A second goal of this code, besides determining execution time on a large dataset, was to give the reader an example of how to import data into ECL’s ML learning algorithms.

This code does the following:

1. Converts the fields of the dataset from string to the field types of the covtype_rec record layout (lines 28 through 210).

2. Makes the dataset suitable for Logistic by making sure the dependent variable/field, CoverType, has only 2 values: 0 or 1 (line 217). It does this by changing CoverType to 1 if the original record had CoverType=2 and CoverType to 0 if the original record had CoverType as something other than 2.

3. Separates the independent variables/fields from the dependent variables/fields(lines 226 through 237).

4. Converts both the independent and dependent datasets to the record format and data type used by ML.Classify.Logistic (lines 243 through 246). Both the independent data (train_indep_Data) and dependent data (train_dep_Data) are converted in two steps: first each is converted to the ML.Types.NumericField data type using the ML macro, ToField. Then, each is discretized using ML.Discretize.ByRounding.

5. Does Logistic Regression learning to produce a model and OUTPUTs the model to the workunit (lines 252 through 256).
